## Quick Reference Commands

### OpenSSL Certificate Operations

```bash
# View certificate details
openssl x509 -in cert.pem -text -noout

# Check certificate expiration
openssl x509 -in cert.pem -noout -dates

# Verify certificate chain
openssl verify -CAfile chain.pem cert.pem

# Check what certificate a server is using
echo | openssl s_client -connect example.com:443 2>/dev/null | \
  openssl x509 -noout -subject -dates

# Extract certificate from server
echo | openssl s_client -connect example.com:443 2>/dev/null | \
  openssl x509 > server-cert.pem

# Generate self-signed certificate (development only!)
openssl req -x509 -newkey rsa:2048 -nodes \
  -keyout key.pem -out cert.pem -days 365

# Check certificate and key match
openssl x509 -noout -modulus -in cert.pem | openssl md5
openssl rsa -noout -modulus -in key.pem | openssl md5
# (hashes should match)

# Convert certificate formats
# PEM to DER
openssl x509 -in cert.pem -outform DER -out cert.der
# DER to PEM
openssl x509 -in cert.der -inform DER -out cert.pem

# View certificate from website in browser
# Chrome: Click padlock ‚Üí Certificate ‚Üí Details
```

## Interview Quick Points

### Must Know ‚≠ê

1. **Chain of Trust**: Root ‚Üí Intermediate ‚Üí Server
2. **Self-signed = Not Trusted** (no CA signature)
3. **SANs** allow one cert for multiple domains
4. **Wildcard** covers *.example.com (one level only)
5. **OCSP Stapling** is best for revocation checking
6. **Private key** must stay private! Never share!
7. **Shorter validity** is better (90 days trend)

### Common Interview Questions

**Q: What's in a certificate?**
- Public key, domain, issuer, validity dates, signature

**Q: Why use intermediate CAs?**
- Keep root CA offline/secure, easier revocation

**Q: Self-signed vs CA-signed?**
- Self-signed: Not trusted by browsers
- CA-signed: Trusted because CA is in trust store

**Q: How to check if cert is valid?**
- Expiration, domain match, chain, revocation, signature
